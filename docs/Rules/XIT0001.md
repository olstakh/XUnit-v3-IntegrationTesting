# XIT0001 - Class-level TestCaseOrderer should be DependencyAwareTestCaseOrderer

Source: [AttributeUsageTestCaseOrdererAnalyzer.cs](../../src/Xunit.v3.IntegrationTesting.Analyzers/Analyzers/AttributeUsageTestCaseOrdererAnalyzer.cs)

|Property|Value|
|--|--|
|Id|XIT0001|
|Category|Usage|
|Severity|<span title='Warning'>⚠️</span>|
|Enabled|✔️|
|Code fix|❌|

A method uses `[FactDependsOn]` (or `[TheoryDependsOn]`), but the class-level `[TestCaseOrderer]` is not `DependencyAwareTestCaseOrderer`. Without the correct orderer, tests will not be ordered according to their declared dependencies.

## Violation

```csharp
[TestCaseOrderer(typeof(SomeOtherOrderer))] // XIT0001
public class MyTests
{
    [FactDependsOn(Dependencies = [nameof(Test2)])]
    public void Test1() { }

    [FactDependsOn]
    public void Test2() { }
}
```

## How to fix

Set the class-level `[TestCaseOrderer]` to `DependencyAwareTestCaseOrderer`:

```csharp
[TestCaseOrderer(typeof(DependencyAwareTestCaseOrderer))]
public class MyTests
{
    [FactDependsOn(Dependencies = [nameof(Test2)])]
    public void Test1() { }

    [FactDependsOn]
    public void Test2() { }
}
```

or remove class-level test case orderer and delegate it to the assembly-level one provided by the MSBuild property `<UseDependencyAwareTestCaseOrderer>true</UseDependencyAwareTestCaseOrderer>` (which is _ON_ by default). Another option is to inherit from `DependencyAwareTestCaseOrderer` which allows overriding the order of tests that don't affect declared topological order (i.e. tests on the same level of dependency)

## When to suppress
Not recommended to suppress. Unless you're sure current orderer respects dependencies (i.e. custom implementation that takes into account `DependsOnCollectionsAttribute`)

See [How to suppress analyzer warnings](../Suppress-warnings.md) for suppression options.
